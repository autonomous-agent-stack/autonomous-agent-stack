## 📖 方案列表

### 1. [单机部署](./01-standalone-deployment.md)
**适合:** 开发测试、小型项目、快速验证

**特点:**
- ✅ 部署简单快速
- ✅ 成本最低
- ✅ 运维复杂度低
- ⚠️ 单点故障风险

**架构:** Nginx → App → PostgreSQL + Redis
**成本:** $10-50/月

### 2. [容器化部署](./02-container-deployment.md)
**适合:** 中小型项目、微服务架构

**特点:**
- ✅ 环境一致性
- ✅ 快速部署
- ✅ 资源隔离
- ✅ 易于扩展

**架构:** Docker Compose 多容器编排
**成本:** $50-200/月

### 3. [Kubernetes 部署](./03-kubernetes-deployment.md)
**适合:** 大型项目、企业级应用

**特点:**
- ✅ 自动扩缩容
- ✅ 自愈能力
- ✅ 滚动更新
- ✅ 服务发现

**架构:** K8s Cluster + HPA + Service Mesh
**成本:** $200-1,000/月

### 4. [云服务部署](./04-cloud-deployment.md)
**适合:** 需要快速扩展、全球部署

**包含:**
- AWS 部署 (ECS/EKS/SageMaker)
- Azure 部署 (AKS/Azure ML)
- GCP 部署 (GKE/Vertex AI)
- 阿里云部署 (ACK/PAI)
- 腾讯云部署 (TKE/TI 平台)

**特点:**
- ✅ 按需付费
- ✅ 全球基础设施
- ✅ 托管服务
- ✅ 快速部署

**成本:** $500-5,000/月

### 5. [混合部署](./05-hybrid-deployment.md)
**适合:** 数据敏感、合规要求

**包含:**
- 公有云 + 私有云
- 边缘计算
- 多云部署
- 容灾方案

**特点:**
- ✅ 数据主权
- ✅ 灵活性高
- ✅ 容灾能力
- ⚠️ 复杂度高

**成本:** $1,000-10,000/月

### 6. [高可用部署](./06-high-availability-deployment.md)
**适合:** 关键业务、7x24 服务

**特点:**
- ✅ 99.99% 可用性
- ✅ 多可用区部署
- ✅ 自动故障转移
- ✅ 数据实时备份

**架构:** 多 AZ + 多主数据库 + 自动故障转移
**成本:** $2,000-20,000/月

### 7. [高性能部署](./07-high-performance-deployment.md)
**适合:** AI 推理、实时处理

**特点:**
- ✅ GPU 加速
- ✅ 分布式部署
- ✅ 缓存优化
- ✅ 网络优化

**架构:** GPU Cluster + Model Sharding + Edge Cache
**成本:** $1,000-10,000/月

### 8. [安全部署](./08-security-deployment.md)
**适合:** 金融、医疗、政府

**特点:**
- ✅ 网络隔离
- ✅ 数据加密
- ✅ 访问控制
- ✅ 审计日志

**架构:** VPC 隔离 + KMS 加密 + IAM + WAF
**成本:** $2,000-15,000/月

### 9. [成本优化部署](./09-cost-optimization-deployment.md)
**适合:** 初创公司、预算敏感

**特点:**
- ✅ Spot 实例 (节省 70-90%)
- ✅ 预留实例 (节省 50-70%)
- ✅ 自动扩缩容
- ✅ 资源优化

**架构:** Spot Fleet + Auto Scaling + Reserved Instances
**成本:** $100-500/月 (比标准方案节省 50-90%)

### 10. [边缘部署](./10-edge-deployment.md)
**适合:** 全球用户、IoT 设备

**特点:**
- ✅ 边缘节点
- ✅ 数据同步
- ✅ 离线支持
- ✅ 就近接入

**架构:** Global Edge + CDN + Edge Functions
**成本:** $1,000-8,000/月

## 💡 最佳实践

### 通用原则

1. **渐进式部署**: 从简单开始,逐步演进
2. **自动化优先**: 自动化一切可自动化的操作
3. **监控先行**: 先建监控,再部署服务
4. **备份重要**: 定期备份,定期演练恢复
5. **文档齐全**: 记录所有配置和变更

### 安全建议

1. **最小权限**: 所有组件使用最小必要权限
2. **加密传输**: 全部使用 HTTPS/TLS
3. **密钥管理**: 使用 KMS/Secret Manager
4. **网络隔离**: VPC/Subnet 隔离
5. **审计日志**: 记录所有操作

### 成本优化

1. **按需使用**: 使用自动扩缩容
2. **预留实例**: 长期工作负载使用预留实例
3. **Spot 实例**: 无状态服务使用 Spot
4. **存储优化**: 冷数据使用低成本存储
5. **定期审查**: 定期审查和优化资源
