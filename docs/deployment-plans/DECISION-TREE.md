## 第一层: 用户规模

### Q1: 你的预期用户规模是多少?

```
< 100 用户
    ↓
【单机部署】(01-standalone-deployment.md)
✅ 部署简单
✅ 成本最低
⚠️ 扩展能力有限
```

```
100 - 1,000 用户
    ↓
Q2: 需要快速扩展吗?
├─ 是 → 【容器化部署】(02-container-deployment.md)
└─ 否 → 【单机部署】(01-standalone-deployment.md)
```

```
1,000 - 10,000 用户
    ↓
Q3: 有运维团队吗?
├─ 是 → 【Kubernetes 部署】(03-kubernetes-deployment.md)
└─ 否 → 【容器化部署】(02-container-deployment.md)
```

```
> 10,000 用户
    ↓

├─ AWS → 【AWS 部署】(04-cloud-deployment.md#aws-部署)
├─ Azure → 【Azure 部署】(04-cloud-deployment.md#azure-部署)
├─ GCP → 【GCP 部署】(04-cloud-deployment.md#gcp-部署)
├─ 阿里云 → 【阿里云部署】(04-cloud-deployment.md#阿里云部署)
├─ 腾讯云 → 【腾讯云部署】(04-cloud-deployment.md#腾讯云部署)
└─ 自建 → 【Kubernetes 部署】(03-kubernetes-deployment.md) + 【高可用部署】(06-high-availability-deployment.md)
```

## 💡 专家建议

### 何时升级?

| 当前方案 | 升级信号 | 升级到 |
|---------|---------|--------|
| 单机部署 | CPU > 80%, 内存 > 85% | 容器化部署 |
| 容器化部署 | 频繁手动扩容 | Kubernetes 部署 |
| Kubernetes 部署 | 跨可用区需求 | 高可用部署 |
| 云服务部署 | 供应商锁定风险 | 多云部署 |
| 单云部署 | 合规要求 | 混合部署 |

### 成本优化技巧

1. **使用预留实例**: 长期工作负载节省 50-70%
2. **Spot 实例**: 无状态服务节省 70-90%
3. **自动扩缩容**: 非高峰期缩减资源
4. **存储分层**: 冷数据使用低成本存储

### 监控指标

每个方案都应该监控:

```yaml
核心指标:
  - 可用性 (Uptime)
  - 响应时间 (Response Time)
  - 错误率 (Error Rate)
  - 吞吐量 (Throughput)

业务指标:
  - 活跃用户数
  - API 调用量
  - 资源使用率
  - 成本效率
```
