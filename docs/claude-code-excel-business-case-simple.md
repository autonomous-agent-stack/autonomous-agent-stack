# AI 自动化销售统计与提成发放
## — 给业务管理者的说明文档

---

## 一、这是什么？

这是一套用 AI 来**自动化处理 Excel 表格**的解决方案，专门用于：

- 📊 销售统计汇总
- 💰 提成计算发放
- 🧾 多平台订单合并
- ⚠️ 异常数据识别

**核心原则**：数字必须准确、可复核、可追溯。

---

## 二、为什么需要这套方案？

### 现状痛点

| 问题 | 影响 |
|------|------|
| 每月手动复制粘贴，容易出错 | 财务数据不准确 |
| 不同平台导出的表格格式不一样 | 需要人工整理，耗时 |
| 提成规则复杂，手动计算容易漏 | 员工投诉、核算困难 |
| 发现问题时难以追溯 | 不知道哪一步出错 |
| 规则变更后很难验证 | 担心影响员工收入 |

### 我们的解决方案

```
原来：人工汇总 3-5 天 → 容易出错 → 难以追溯
现在：AI 自动处理 → 准确稳定 → 完整审计记录
```

## 四、销售统计与提成计算流程

### 输入文件

每月需要处理的 4 类表格：

```
┌─────────────────────────────────────────────────────────┐
│  1. 平台订单明细表    │  来自各销售平台导出              │
│  2. 销售跟单汇总表    │  团队内部整理                   │
│  3. 退款/调整表       │  财务提供的调整记录             │
│  4. 提成规则表        │  HR 或管理层提供的规则          │
└─────────────────────────────────────────────────────────┘
```

### 自动处理流程

```
步骤 1: 读取识别
   ↓
步骤 2: 数据清洗（统一格式、去重、校验）
   ↓
步骤 3: 按规则计算销售额、回款额、有效订单
   ↓
步骤 4: 按岗位/区域/渠道计算提成
   ↓
步骤 5: 生成异常报告
   ↓
步骤 6: 输出结果供人工审核
```

### 输出结果

系统会自动生成 4 份文件：

| 文件 | 用途 |
|------|------|
| 标准化订单明细 | 便于后续分析 |
| 异常清单 | 标记需要人工确认的数据 |
| 销售统计汇总表 | 给管理层的经营数据 |
| 提成发放表 | 给财务的最终发放依据 |

## 五、数据安全保障

### 三层验证机制

```
第一层：自动校验
   └── 检查数据完整性、格式正确性

第二层：规则验证
   └── 按设定的规则重新计算，确保无误

第三层：人工审核
   └── 关键数据点人工抽查确认
```

### 完整审计记录

每次处理都会保存：

- ✅ 使用了哪个版本的规则
- ✅ 处理了哪些文件
- ✅ 生成了什么结果
- ✅ 谁在什么时候审核过
- ✅ 发现了哪些异常

**发现问题可以随时追溯、重新计算。**

## 六、AI 在这里的角色

### AI 帮你做的事

| 日常维护 | 业务扩展 |
|---------|---------|
| 新增一种表格格式 | 新增一套提成规则 |
| 修复数据映射问题 | 优化计算逻辑 |
| 补充测试案例 | 生成异常解释报告 |
| 更新操作文档 | 制作管理摘要 |

### AI 不做的事

- ❌ 不直接自由计算最终数字
- ❌ 不在同一时间对同一份表给出不同结果
- ❌ 不在无人工确认的情况下修改发放数据
- ❌ 不跳过关键审核点

**核心原则：AI 负责写程序和维护程序，程序负责算数字。**

## 十、常见问题

**Q: AI 算的钱会出错吗？**

A: AI 不直接算钱。AI 负责编写和维护计算程序，程序按照固定规则计算。每次计算都可以验证。

**Q: 规则变了怎么办？**

A: AI 可以快速修改规则，并用历史数据验证新旧规则的差异，人工确认后生效。

**Q: 发现错误能追溯吗？**

A: 能。系统保存完整的操作记录和审计日志，可以追溯到每一次计算的详细过程。

**Q: 多个销售团队规则不同能处理吗？**

A: 能。系统支持多套规则并行运行，自动按团队匹配对应规则。

**Q: 需要什么投入？**

A: 初期需要提供真实表格样本和业务规则说明，系统会自动学习。后期主要在规则变更和异常处理时需要人工参与。
