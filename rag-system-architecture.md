## 性能优化点

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          性能优化策略                                        │
└─────────────────────────────────────────────────────────────────────────────┘

1. 查询阶段优化
   ├─ 缓存常见查询 (LRU Cache)
   ├─ 并行处理查询扩展 (ThreadPool)
   └─ 批量查询处理 (Batch Inference)

2. 检索阶段优化
   ├─ 使用FAISS加速向量检索 (GPU可选)
   ├─ 实现向量索引 (IVF/PQ)
   ├─ 分片检索 (Sharding)
   └─ 稀疏检索优化 (BM25优化)

3. 重排序优化
   ├─ 只对Top-K重排序 (减少计算)
   ├─ 使用轻量模型 (bge-reranker-base)
   ├─ 缓存重排序结果 (Query Cache)
   └─ 级联重排序 (粗排→精排)

4. 上下文优化
   ├─ 智能压缩 (减少Token)
   ├─ 上下文去重 (去除冗余)
   ├─ 分段处理 (长上下文)
   └─ 选择性保留 (相关性过滤)

5. 系统优化
   ├─ 异步处理 (Async IO)
   ├─ 连接池 (Database Pool)
   ├─ 负载均衡 (Load Balance)
   └─ 监控告警 (Metrics & Alert)
```
