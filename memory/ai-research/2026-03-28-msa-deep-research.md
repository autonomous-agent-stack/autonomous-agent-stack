## 🔬 技术核心

### 1. Memory Sparse Attention (MSA)

**核心架构**：
- **端到端可训练**：检索和生成集成在单一可微分循环中
- **可扩展稀疏注意力**：O(L) 复杂度
- **文档级 RoPE**：每个文档重置位置，防止位置漂移
- **精度损失**：<9%（16K→100M Token）

**三大核心模块**：

#### 1.1 可扩展稀疏注意力
- **Top-k 选择** + **稀疏注意力**
- **端到端可微分**
- **文档解耦**（推理时）

#### 1.2 KV Cache 压缩 + Memory Parallel
- **分层存储**：
  - GPU 驻留路由键（Kᵣ）
  - CPU 内容 K/V
- **分布式评分**
- **按需传输**
- **硬件需求**：2×A800（100M Token 推理）

#### 1.3 Memory Interleave
- **自适应交替**：
  - 生成式检索 → 上下文扩展 → 生成
- **多跳推理**：显著提升跨文档推理能力

## 📈 性能数据

### 1. 长上下文 QA（9 个数据集）

**对比对象**：
- 同骨架 RAG（Qwen3-4B）
- SOTA RAG 栈（KaLMv2 + 大型生成器）

**核心结果**（平均分）：
- **MSA**: 3.760
- Qwen3-4B RAG: 3.242（+16.0%）
- Qwen3-4B RAG+rerank: 3.372（+11.5%）
- HippoRAG2: 3.275（+14.8%）
- KaLMv2+Qwen3-235B: 3.506（+7.2%）
- KaLMv2+Llama-3.3-70B: 3.396（+10.7%）

**结论**：
- ✅ 4B 参数模型超越 235B 方案
- ✅ 在 4/9 数据集上取得最佳成绩
- ✅ 平均提升 5.0%-16.0%

---

### 2. NIAH (Needle-in-a-Haystack) 稳定性

**测试范围**：32K → 1M Token

**核心结果**：
- **MSA @1M**: 94.84% 准确率
- **原始骨架 @1M**: 24.69%（崩溃）
- **混合线性注意力模型**: ≥128K/256K 明显退化
- **外部记忆 Agent (RL-MemoryAgent-14B)**: 稳定但绝对准确率较低

**结论**：
- ✅ MSA 在 1M Token 保持高准确率
- ✅ 显著优于其他长上下文方案
- ✅ 稳定性和准确性兼得

---

### 3. 扩展性（16K→100M Token）

**核心数据**：
- **16K → 100M Token**: <9% 精度损失
- **训练数据**: 158.95B Token
- **训练策略**:
  - 连续预训练（辅助路由损失）
  - 两阶段 SFT（8k→64k 课程）

**扩展曲线**：
- 线性扩展（接近 O(L)）
- 无明显性能崩溃点
- 推理成本可控
