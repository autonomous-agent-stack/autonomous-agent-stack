#### 2.2 设计技能

**技能设计原则**:
1. **单一职责** - 每个技能只做一件事
2. **清晰接口** - 输入输出明确
3. **错误处理** - 异常情况处理
4. **可测试性** - 易于单元测试

**技能模板**:
```python
def skill_name(input: str) -> Dict[str, Any]:
    """
    技能描述
    
    Args:
        input: 输入说明
    
    Returns:
        {
            "success": bool,
            "data": any,
            "error": str (optional)
        }
    """
    try:
        # 实现逻辑
        result = ...
        
        return {
            "success": True,
            "data": result
        }
    except Exception as e:
        return {
            "success": False,
            "error": str(e)
        }
```

**输出**: skills/*.py
