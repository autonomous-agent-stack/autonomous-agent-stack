```
                 ↓
           更新模型 → 新技能提取
                 ↓
           下次对话使用新技能
```

## 🔄 三种运行模式

### 1. skills_only（轻量级）

**特点**：
- 无需 GPU
- 只需 LLM API
- 技能注入 + 自动总结

**流程**：
```
对话 → Proxy → 技能注入 → LLM → 响应
         ↓
    会话结束 → 自动总结技能
         ↓
    下次对话使用新技能
```

**适用场景**：
- 没有 GPU 的团队
- 想快速体验智能技能注入
- 不需要模型训练

---

### 2. rl（标准 RL）

**特点**：
- skills_only + RL 训练
- 训练立即执行（batch 满时）
- 需要 GPU/云训练后端

**流程**：
```
对话 → Proxy → 技能注入 → LLM → 响应
         ↓
    记录交互数据
         ↓
    PRM 打分（异步）
         ↓
    batch 满 → 立即训练
         ↓
    更新模型
```

**适用场景**：
- 有 GPU 的团队
- 想持续优化模型
- 能接受训练中断服务

---

### 3. madmax（默认，推荐）

**特点**：
- skills_only + RL + 智能调度器
- 训练只在空闲时间执行
- 不中断服务

**调度策略**：
- **睡眠时间**：23:00-07:00
- **空闲时间**：无活动 > 30 分钟
- **会议时间**：Google Calendar 集成

**流程**：
```
对话 → Proxy → 技能注入 → LLM → 响应
         ↓
    记录交互数据
         ↓
    PRM 打分（异步）
         ↓
    累积经验（不立即训练）
         ↓
    检测空闲时间
         ↓
    空闲 → 开始训练
         ↓
    更新模型
```

**适用场景**：
- 生产环境
- 不能中断服务
- 想持续优化

## 🧠 自演化机制

### 1. 技能注入（Skill Injection）

**原理**：
- Proxy 拦截每轮对话
- 从技能库检索相关技能（top-k）
- 注入到 prompt 中
- LLM 基于技能生成响应

**技能格式**：
```markdown
# Skill: API Bug Fix

When fixing API bugs:
1. Reproduce the issue
2. Check error logs
3. Identify root cause
4. Write test case
5. Fix and verify

Example:
- Error: 404 Not Found
- Cause: Missing route
- Fix: Add route handler
```

**检索策略**：
- `template`: 关键词匹配
- `embedding`: 语义检索

---

### 2. 技能进化（Skill Evolution）

**原理**：
- 会话结束后，分析失败案例
- 提取新的技能模式
- 保存到技能库
- 下次对话使用

**流程**：
```
会话结束 → 分析对话记录
         ↓
    识别失败模式
         ↓
    提取通用规则
         ↓
    生成新技能（LLM）
         ↓
    保存到技能库
         ↓
    下次对话可用
```

**示例**：
```python
# 失败案例
用户: "帮我部署到 AWS"
Agent: "我需要访问权限..."
结果: 失败（没有权限检查）

# 提取新技能
Skill: AWS Deployment Checklist
1. Check IAM permissions
2. Verify VPC configuration
3. Test S3 access
4. Configure security groups
```

---

### 3. RL 训练（Reinforcement Learning）

**算法**：GRPO（Group Relative Policy Optimization）

**核心组件**：
1. **PRM（Process Reward Model）**：给每个响应打分
2. **LoRA Fine-tuning**：低秩适配训练
3. **Hot-swap**：热切换权重

**训练流程**：
```
累积 batch → PRM 打分 → 计算 reward
         ↓
    GRPO 优化
         ↓
    LoRA 更新
         ↓
    热切换权重
         ↓
    新对话使用新模型
```

**支持后端**：
- **Tinker**（默认）：云 LoRA 训练
- **MinT**：MindLab 研究后端
- **Weaver**：Nex AGI 训练平台

---

### 4. 智能调度器（Scheduler）

**目标**：只在空闲时间训练，不中断服务

**触发条件**：
1. **睡眠时间**：23:00-07:00
2. **空闲时间**：无活动 > 30 分钟
3. **会议时间**：Google Calendar 集成

**调度策略**：
```python
def should_train():
    if is_sleep_time():
        return True
    if is_idle() > 30 minutes:
        return True
    if is_in_meeting():
        return True
    return False
```

**优势**：
- 不影响用户体验
- 充分利用空闲时间
- 自动化训练流程

## 💡 关键创新

### 1. 异步设计

**问题**：传统 RL 训练会阻塞服务

**解决方案**：
```
┌──────────────┐
│  Serving     │ ← 实时响应
└──────────────┘
       ↓ 异步
┌──────────────┐
│  Reward      │ ← 后台打分
└──────────────┘
       ↓ 异步
┌──────────────┐
│  Training    │ ← 空闲训练
└──────────────┘
```

**优势**：
- 服务不中断
- 充分利用资源
- 持续优化

---

### 2. 技能进化闭环

**流程**：
```
对话 → 使用技能 → 记录结果
         ↓
    分析失败案例
         ↓
    提取新技能
         ↓
    下次对话使用
```

**优势**：
- 从失败中学习
- 持续改进技能库
- 无需人工维护

---

### 3. API-first 设计

**问题**：传统 RL 需要本地 GPU

**解决方案**：
- 使用云训练 API（Tinker/MinT/Weaver）
- 无需本地基础设施
- 按需付费

**优势**：
- 降低门槛
- 弹性扩展
- 成本可控
