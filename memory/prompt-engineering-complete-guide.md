## 📝 提示词结构模板

### **1. 基础结构（5要素）**

```markdown
## 背景
<项目/任务背景>

## 目标
<具体要做什么>

## 约束
<限制条件>

## 要求
<具体要求>

## 输出格式
<期望的输出格式>
```

---

### **2. 完整示例**

#### **❌ 差的提示词**
```
写个函数
```

**问题**:
- 无背景
- 无目标
- 无约束
- 无要求
- 无格式

---

#### **✅ 好的提示词**
```markdown
## 背景
电商平台后端服务，使用 FastAPI + PostgreSQL

## 目标
创建用户注册和登录功能

## 约束
- 使用 JWT 认证
- 密码加密存储（bcrypt）
- 邮箱格式验证
- 遵循 RESTful API 设计

## 要求
1. 完整的错误处理
2. 单元测试覆盖
3. API 文档
4. 代码注释

## 输出格式
- 代码文件（带注释）
- 测试文件
- API 文档（Markdown）
```

## 🎯 提示词分类

### **1. 按任务类型分类**

#### **1.1 代码生成**
```markdown
## 任务类型: 代码生成

### 提示词模板
创建一个[功能名称]：
- 功能描述：[详细描述]
- 技术栈：[技术]
- 要求：[具体要求]
```

**示例**:
```markdown
创建一个用户认证系统：
- 功能描述：JWT认证 + 密码加密 + 邮箱验证
- 技术栈：FastAPI + PostgreSQL
- 要求：完整错误处理 + 单元测试
```

---

#### **1.2 代码审查**
```markdown
## 任务类型: 代码审查

### 提示词模板
审查以下代码：
[代码]

重点检查：
- [检查项1]
- [检查项2]
```

**示例**:
```markdown
审查以下代码：
def authenticate(email, password):
    user = db.query(email)
    if user and user.password == password:
        return token
    return None

重点检查：
- 安全性（密码加密）
- 错误处理
- 性能
```

---

#### **1.3 代码优化**
```markdown
## 任务类型: 代码优化

### 提示词模板
优化以下代码：
[代码]

优化目标：
- [目标1]
- [目标2]
```

**示例**:
```markdown
优化以下代码：
for i in range(1000):
    result = process(data[i])

优化目标：
- 性能（减少时间）
- 内存（减少使用）
```

## 💡 提示词技巧

### **1. 上下文提供**

#### **技巧1.1: 明确技术栈**
```markdown
❌ 差
写个API

✅ 好
创建一个REST API：
- 框架：FastAPI
- 数据库：PostgreSQL
- 缓存：Redis
```

---

#### **技巧1.2: 提供示例代码**
```markdown
创建一个类似的函数：

示例代码：
def example_function():
    # 好的实现
    return result

要求：
- 遵循示例的风格
- 保持代码质量
```

---

#### **技巧1.3: 指定编码风格**
```markdown
创建一个函数：
- 编码风格：Google Python Style Guide
- 类型注解：必须有
- 文档字符串：必须有
```

---

### **2. 约束条件**

#### **技巧2.1: 性能约束**
```markdown
优化代码：
- 性能要求：处理时间 <100ms
- 内存限制：<50MB
- 并发：支持1000 QPS
```

---

#### **技巧2.2: 安全约束**
```markdown
创建认证系统：
- 密码：bcrypt加密
- Token：JWT，24小时过期
- HTTPS：强制HTTPS
```

---

#### **技巧2.3: 兼容性约束**
```markdown
修改代码：
- 保持向后兼容
- 不修改公共API
- 保留现有测试
```

---

### **3. 迭代优化**

#### **技巧3.1: 逐步细化**
```markdown
# 第1轮：基础版本
创建用户API

# 第2轮：添加验证
添加邮箱格式验证

# 第3轮：添加测试
添加单元测试

# 第4轮：优化性能
优化数据库查询
```

---

#### **技巧3.2: 反馈循环**
```markdown
生成代码后，如果不符合要求：
- 描述问题
- 提供具体修改建议
- 给出示例

例如：
代码有问题：
- 问题：密码没有加密
- 修改：使用bcrypt
- 示例：hashed = bcrypt.hashpw(password, salt)
```
