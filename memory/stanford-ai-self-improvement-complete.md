## 🎯 核心定义

### **持续自我提升式AI**
- **定义**: 一旦被创造，能自主且持续自我改进，改进效果优于人类创造者
- **两个前提假设**:
  1. **参数化** - 基于神经网络，知识编码在参数权重
  2. **预训练** - 经历资源密集型预训练阶段

### **三个核心特性**
1. 持续获取新知识，不灾难性遗忘已有能力
2. 自主生成训练信号，超越人类生成信号
3. 自主设计学习算法，适配自生成信号

---

## 🔍 三大局限性（当前AI天花板）

### **1. 训练后权重静态化**
- 模型部署后参数固定
- 上下文压缩有损（遗忘）
- 缺乏持续学习能力
- **对比**: 人类即使过了很久，依然能隐约记起关键信息

### **2. 人类数据有限性**
- **关键图表**:
  - 左边：语言模型缩放定律（token数 vs 测试损失）
  - 右边：前沿模型token数逼近互联网总量
- **矛盾**: 人类数据有限 vs 需求无限
- **本质**: 不可调和的矛盾

### **3. 算法发现依赖人力**
- 人类研发：产生想法 → 实验验证 → 形成成果
- **成本**: 高昂且效率极低
- **现状**: 只发现了算法子集（梯度下降、Transformer等）
- **问题**: 大量优秀算法因人类认知和精力限制未被发现

## 🚀 三个研究方向

### **方向1: 持续知识获取（合成持续训练）**

#### **问题**
如何让模型在预训练后，持续学习小众领域知识，不遗忘已有能力？

#### **案例对比**
| 领域 | 数据量 | 模型表现 |
|------|--------|----------|
| **线性代数** | 丰富（教科书、讲义、习题、代码） | 扎实 |
| **小众领域**（新GitHub库） | 稀疏 | 无法理解 |

#### **实验设计**
- **Quality数据集**: 265本专业书籍，180万token
- **测试**: 4000个多项选择题，闭卷测试（无上下文）

#### **基准结果**
| 模型 | 准确率 |
|------|--------|
| Llama 3 8B（基础） | 39.49% |
| Llama 3 8B（原始书籍微调） | 38.15%（下降） |
| GPT-3.5 | 44.81% |
| GPT-4 | 51.30% |

**问题**: 原始数据token太少，训练无法优化，导致遗忘

#### **解决方案演进**

**方案1: 简单重写**
- 提示词: "像维基百科文章一样重写这本书"
- **结果**: 提升斜率平缓，未超过GPT-4
- **问题**: 固定提示词 → 缺乏多样性 → 简单复述

**方案2: 实体图合成数据生成** ⭐
- **核心**: 两阶段流程
  1. **提取实体**: 从源文档提取核心实体（如牙科书籍→氟化物、牙釉质、牙医）
  2. **随机组合**: 抽取部分实体，描述关系（如"在预防蛀牙背景下，氟化物和牙釉质的相互作用"）
- **优势**:
  - 实体改变 → 词向量变化 → 非线性输出变化
  - 多样化训练语料
- **结果**: 准确率56.22%（接近GPT-4）

**方案3: 合成持续训练 + 开卷问答**
- **结果**: 准确率62.60%（开卷基准60.35%）
- **结论**: 合成训练 + 检索工具 = 互补（1+1>2）

**未来应用**: 开源模型适配个性化场景的核心范式

### **方向2: 预训练能力自我提升（合成引导预训练 SBP）**

#### **核心发现**
- OpenAI o1 发布时，很多人认为强大推理源于海量人工标注
- **实验**: 仅1万条思维链数据监督微调 → 达到o1 preview级别
- **结论**: 后训练能力 = 预训练知识的泛化
- **关键**: 预训练才是模型能力的主心骨

#### **思想实验**
- **假设**: 世界只有A、B、C、D、E五个token
- **训练**: 随机组合文档
- **结果**: Transformer无法学到任何有意义知识（初始概率20% = 训练分布）
- **启示**: 模型能学习，因为自然语言存在结构性相关性

#### **两种视角**
1. **统计学视角**: token之间存在统计相关性
2. **计算学视角**: 自然语言存在可压缩模式（信源编码）

#### **核心洞察**
**当前预训练遗漏**: 文档之间的相关性

**案例**:
- 《哈利·波特与魔法石》 → 三年后电影剧本（强相关性）
- 《Attention is All You Need》 → GPT-2代码库第91行（直接相关性）

**问题**: 当前预训练单独处理每篇文档，忽略跨文档相关性

#### **SBP 三步流程**

**避免质疑**: 不是从教师模型蒸馏知识

**流程**:
1. **固定真实数据预训练** - 从头预训练语言模型
2. **合成数据生成器** - 微调模型为生成器（不引入新真实文本）
3. **混合预训练** - 真实数据 + 合成数据，观察性能提升

**成功标准**: 不需人类提供更多数据/算法，自己提升自己

#### **技术实现**

**步骤1: 最近邻配对**
- 使用DCLM数据集子集
- 通义千问0.6B嵌入模型 → 词向量
- 根据相似度配对
  - Transformer论文 ↔ PyTorch注意力代码
  - 哈利·波特书籍 ↔ 电影评论

**步骤2: 合成微调**
- 以预训练模型为初始化
- 监督微调目标：关联图一侧文档为条件 → 最大化另一侧对数概率
- **关键**: 一个源文档 → 多个目标文档（增加熵，学习复杂关联）

**步骤3: 规模化合成**
- 生成温度 = 1
- 不使用复杂提示词技巧
- 对所有真实文档反复生成
- **目的**: 保证多样性
  - 小众小说 → 评论、总结、相关分析
  - 咖啡馆推荐 → 浓缩咖啡机、咖啡豆选择（原始未提及）

#### **实验结果**

**三组对比**:
- **基准组**: 重复使用固定真实数据
- **SBP组**: 相同真实数据 + 计算量，用合成数据替代重复
- **Oracle组**: 无限真实数据，相同计算量

**训练曲线**:
- **初期**: Oracle ≈ 基准 < SBP
- **中期**: 基准饱和（反复学习相同数据，无新知识），SBP和Oracle持续线性下降

**定量结果**:

| 规模 | 基准 | SBP提升 | Oracle提升 |
|------|------|---------|-----------|
| 200B token, 3B参数 | - | +2.32% | +5.54% |
| 1T token, 6B参数 | - | +1.32% | - |

**质量提升**:
- 200B规模: 合成数据非事实错误率 15.1%
- 1T规模: 错误率降至 6.5%（真实数据 1.8%）
- **结论**: 模型规模越大 → 知识越扎实 → 合成数据越准确

**正向循环**: 模型越强 → 合成数据质量越高 → 模型更强

### **方向3: AI设计AI（自动化AI研究员）**

#### **哲学基础**
**AI为什么能在AI研究领域做得比人类更好？**

- **科学进步**（罗纳德·费希尔）: 提出假设 → 实验证伪
- **AI优势**:
  - 提出假设 = 生成文本（语言模型强项）
  - 实验证伪 = 编写代码（计算机强项）
- **数据**: SWE-Bench准确率 20% → 80%（AI编写调试代码能力强大）

#### **研究环境抽象**

**两个核心组件**:
1. **上下文**（描述任务）= 代码库
2. **价值函数**（评估优劣）= 实验结果

**两个具体环境**:

| 环境 | 代码库 | 资源 | 评估指标 |
|------|--------|------|----------|
| **预训练环境** | GPT-2预训练Python脚本 | 8张H100 GPU | 达到测试损失3.28所需时间 |
| **后训练环境** | GSM8K训练GRPO算法 | 1张Blackwell GPU | MATH500测试准确率 |

#### **自动化AI研究员工作流程**

**四步循环**:
1. **构思器** → 根据上下文生成算法想法
2. **执行器** → 将想法转化为代码diff
3. **价值函数** → 输入研究环境，得到性能评估
4. **学习模块** → 从结果总结经验，优化构思器

#### **迭代测试时搜索**

**想法库**: 记录所有过去实验想法和结果

**生成新想法时**:
1. **Exploit**（利用）: 结合高价值想法优势（融合多个好算法）
2. **Explore**（探索）: 生成完全不同思路（避免局部最优）

#### **实验结果**

**后训练任务（数学推理）**:
- 基准准确率: 48.0%
- AI优化后: 69.4%
- **人类专家**: 68.8%
- **结论**: 特定任务上，AI已超越人类

**预训练任务**:
- 基准完成时间: 36.9分钟
- AI优化后: 19.7分钟
- 人类专家顶尖: 2.1分钟
- **结论**: 实现近一半效率提升

#### **定性结果：数学工作记忆模拟**

**AI自主提出的想法**:
- 维护上下文缓冲区（数学事实、定义、中间结果）
- 模拟人类工作记忆过程
- 随解题推进，缓冲区不断更新

**AI完整实现**:
- 定义`MathContextBuffer`类
- 核心方法：添加上下文、获取上下文
- 集成到强化学习训练循环

**结果**: 数学推理准确率提升10%

**杨紫童反思**:
- 他自己也有类似手写笔记本（裂项相消、琴生不等式等技巧）
- AI自主提出想法与人类经验不谋而合
- **结论**: AI已能产生人类专家水平的原创性思路

## 💡 哲学思考

### **AI超越人类是必然的**

**不是数量堆叠，而是算法演化**

**当前AI优势（无趣）**:
- 人类数据质量更高，但AI数据无限
- 人类构思能力更强，但AI可运行几万次实验

**真正的超越**:
- 源于算法的自主演化

### **爱因斯坦的例子**

**广义相对论场方程**:
- 未经修改 → 精准预言宇宙膨胀
- 1910年代科学界认为宇宙静止
- 爱因斯坦1917年加入宇宙学常数 → 预测静止宇宙
- 1929年哈勃证实宇宙膨胀
- 爱因斯坦：这是"最大的错误"

**启示**:
- 理论一旦被创造 → 拥有自己的生命力
- 演化出超越创造者认知的内涵
- 场方程编码宇宙膨胀真理，即使爱因斯坦本人一开始也无法接受

**类比**:
- 人类基于算法创造AI → 可能演化出超越人类认知的智能
- 不是因为AI是人类能力子集
- 而是算法演化过程类似爱因斯坦推导场方程
- **本质**: 对客观规律的探索
- **客观规律的深度**: 无限

### **最终观点**

> **我们不该问AI能否超越人类，因为创造物超越创造者，本身就是宇宙演化的常态**
