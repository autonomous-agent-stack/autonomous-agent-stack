### **核心原则（永久记住）**

#### **1. 一个 PR = 一件事**
- 翻译 PR = 只翻译
- 功能 PR = 只功能
- 文档 PR = 只文档

#### **2. 基于 upstream/master**
```bash
# ❌ 错误
git checkout -b branch master

# ✅ 正确
git fetch upstream
git checkout -b branch upstream/master
```

#### **3. 提交前必须检查**
```bash
# 检查文件
git show --name-only

# 检查 commit
git log upstream/master..HEAD --oneline

# 检查 diff
git diff upstream/master --stat
```

### **正确流程（翻译 PR）**
```bash
# 1. 基于 upstream/master
git fetch upstream
git checkout -b translation-branch upstream/master

# 2. 只添加翻译文件
git add README-ZH-CN.md README-ZH-TW.md

# 3. 检查 commit
git show --stat
git show --name-only

# 4. 确认后推送
git push -u origin translation-branch

# 5. 创建 PR
gh pr create --repo original/repo
```
