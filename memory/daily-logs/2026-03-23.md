### MAS Factory 核心发现

#### 核心优势
1. **Vibe Graphing** - 自然语言生成工作流
   - 输入: 自然语言描述（"设计一个软件开发流程"）
   - 处理: GLM-5 生成图结构 → JSON Blueprint → 编译
   - 输出: 可运行的多智能体协作工作流

2. **双模型策略** - 成本暴砍 95%
   - **GLM-5**（规划）: 1 次调用，高成本
   - **GLM-4-Air**（执行）: 100+ 次调用，低成本
   - **总成本**: $0.26（vs $6.08 单模型）

3. **核心组件**
   - **Agent**: 执行单元（CEO, CTO, 程序员）
   - **Graph**: 子图（需求分析阶段）
   - **Loop**: 循环执行（测试循环，最多 3 次）
   - **Switch**: 分支判断（根据条件选择路径）
   - **Human**: 人工介入（代码审查确认）

### 3 种实现方案

#### 方案 1: Vibe Graphing ⭐⭐⭐⭐⭐ (推荐)
- **难度**: ⭐⭐⭐
- **时间**: 30 分钟
- **优势**: 自然语言生成工作流，成本最低
- **适用**: 快速原型，非技术人员
- **示例**:
```python
from masfactory import RootGraph, VibeGraph
from masfactory_config import build_model, invoke_model

g = RootGraph(name="malu_landing_page")
vibe = g.create_node(
    VibeGraph,
    name="vibe_graph",
    invoke_model=invoke_model,
    build_model=build_model,
    build_instructions="设计一个玛露落地页开发工作流",
)
g.build()
result = g.invoke(input={}, attributes={})
```

4. **test_loop**: 测试工程师 + 程序员 测试（最多 3 次）
5. **test_error_summary_phase**: 总结测试错误
6. **test_modification_phase**: 修复代码
