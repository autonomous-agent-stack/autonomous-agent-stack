- Telegram gateway 已接上 `/status` provider/memory 摘要、`/skills` catalog/detail、`/reset` 会话重置，形成最小可用指令面。

  - 新增只读 `/api/v1/approvals` API（health/list/get），当前走 admin bearer 保护。
  - Telegram 已接上 `/approve` 和 `/approve <approval_id>`，先做 pending 列表与详情，不提前做批准动作。
  - `/api/v1/panel/state` 已带 `pending_approvals`，panel view 也增加了“待审批”观察面。
- 审批闭环继续推进：
  - `/api/v1/approvals/{approval_id}/decision` 已落地，admin 高权限可直接 resolve approval。
  - panel 已支持 `/api/v1/panel/approvals`、`/approve`、`/reject`，并把决策结果写回同一 approval store。
  - panel approval 动作现在会写审计日志并触发 Telegram manual action notify，不再只是一块只读观察面。
- admin 控制面继续收口：
  - `/api/v1/admin/approvals`、`/approve`、`/reject` 已补齐并接到 `/api/v1/admin/view` 的 Approval Queue。

  - admin backend 回归已覆盖 capability inventory、approval queue 视图、审批列表过滤与 resolve。

  - Telegram 决策动作直接写回统一 approval store，并写入 `resolved_via=telegram_command` 元数据，不再是只读查询。
  - help 文案和 approval detail 都补了决策用法，减少用户在 bot 内来回试错。
