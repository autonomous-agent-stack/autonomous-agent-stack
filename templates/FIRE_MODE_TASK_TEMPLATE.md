## 🎯 任务分类

### P0 紧急（5 分钟内）
- 提交未推送的文件
- 修复紧急 bug
- 推送到 GitHub

### P1 高优先级（5-15 分钟）
- 补充缺失文档
- 生成分析报告
- 创建自动化工具

### P2 中优先级（15-30 分钟）
- 深度项目分析
- 代码重构
- 测试补充

## ✅ 成功标准

1. **可量化输出**（文件/报告/代码）
2. **真实时间记录**（不夸大）
3. **立即提交**（保持节奏）
4. **有价值的产出**（长期收益）

---

## 🚫 避免的错误

1. **过度承诺时间**（实际 20 分钟说成 30 分钟）
2. **任务过大**（超过 15 分钟）
3. **依赖外部 API**（容易限流）
4. **不提交成果**（失去动力）

## 🎯 任务示例

### 示例 1: 文件提交（2 分钟）

```markdown
### 📋 任务 1: 提交未推送文件（2 分钟）

**目标**: 提交所有未推送的文件

**步骤**:
1. 检查 git status
2. git add .
3. git commit -m "feat: 描述"

**输出**:
- 提交的文件数
- 代码行数

**注意**: 避免提交敏感信息
```

### 示例 2: 报告生成（5 分钟）

```markdown
### 📋 任务 2: 生成项目健康度报告（5 分钟）

**目标**: 生成多项目健康度分析

**步骤**:
1. 扫描所有项目
2. 统计健康度指标
3. 生成报告文件

**输出**:
- 项目数量
- 健康度评分
- 改进建议

**报告位置**: memory/fire-mode-2026-03-27/health-report.md
```
