# 大模型评估基准 (LLM Evaluation Benchmarks)

## 1. MMLU (Massive Multitask Language Understanding)

**概述**
- 测试模型在57个学科领域的知识和推理能力
- 由Brown等人于2020年提出,作为GPT-3论文的一部分

**测试内容**
- 涵盖15个领域:人文学科、STEM、社会科学等
- 57个具体科目:数学、历史、法律、医学、物理、计算机科学等
- 多选题格式(4个选项)

**评估方式**
- 5-shot learning:提供5个示例后让模型回答
- 使用CoT(Chain-of-Thought)可提升性能
- 评估模型的事实知识和推理能力

**代表性分数(2024)**
- GPT-4: ~86%
- Claude 3 Opus: ~86%
- Gemini Ultra: ~83%
- LLaMA 3 70B: ~82%

---

## 2. HellaSwag

**概述**
- 常识推理基准,测试模型完成句子和预测后续场景的能力
- 由Zellers等人于2019年提出

**测试内容**
- 基于in-the-wild视频的场景描述
- 给定场景开头,要求选择最可能的后续
- 评估模型的常识理解和推理能力

**评估方式**
- 从4个选项中选择最合理的后续
- 测试对日常情境的"常识推理"
- 0-shot或5-shot测试

**代表性分数(2024)**
- GPT-4: ~95%
- Claude 3 Opus: ~95%
- LLaMA 3 70B: ~90%

---

## 3. TruthfulQA

**概述**
- 测试模型回答的真实性,避免幻觉(hallucination)
- 由Lin等人于2021年提出

**测试内容**
- 817个问题,涵盖常见误解
- 人类经常给出错误答案的问题
- 评估模型是否模仿人类错误还是提供事实真相

**评估方式**
- 多选题:正确答案 vs 常见的误解
- 同时评估准确性和可信度
- 测试模型的诚实度和事实性

**关键指标**
- Truthfulness:正确答案的比例
- Informative:提供有用信息的程度

**代表性分数(2024)**
- GPT-4: ~60% (truthfulness)
- Claude 3 Opus: ~65%
- 好的模型应该在50%以上

---

## 4. HumanEval

**概述**
- Python代码生成基准,由OpenAI于2021年提出
- 测试模型生成功能性代码的能力

**测试内容**
- 164个手写的Python编程问题
- 每个问题包含函数签名和文档字符串
- 要求模型完成函数实现

**评估方式**
- 生成代码后运行单元测试
- pass@k:前k个预测中有多少能通过测试
- 主要使用pass@1和pass@10

**关键指标**
- pass@1:第一个预测正确的比例
- pass@10:前10个预测中至少有一个正确的比例

**代表性分数(2024)**
- GPT-4: ~91% (pass@1)
- Claude 3 Opus: ~88%
- LLaMA 3 70B: ~81%

---

## 5. GSM8K (Grade School Math 8K)

**概述**
- 小学数学问题基准,由Cobbe等人于2021年提出
- 测试模型的数学推理和计算能力

**测试内容**
- 8,500个小学数学应用题
- 涵盖1-8年级数学知识点
- 需要多步推理和计算

**评估方式**
- 生成完整解答过程和最终答案
- 要求正确的数值答案
- 使用CoT(Chain-of-Thought)显著提升性能

**关键特点**
- 问题长度:平均400词
- 需要多步推理(平均2-3步)
- 测试符号推理和算术计算

**代表性分数(2024)**
- GPT-4: ~92%
- Claude 3 Opus: ~95%
- LLaMA 3 70B: ~93%

## 综合对比

| 基准 | 能力维度 | 难度 | 数据量 | 形式 |
|--------|----------|--------|--------|------|
| MMLU | 知识广度+推理 | 中高 | 15,958题 | 多选 |
| HellaSwag | 常识推理 | 中 | 70,000+ | 多选 |
| TruthfulQA | 真实性/抗幻觉 | 高 | 817题 | 多选+生成 |
| HumanEval | 代码生成 | 高 | 164题 | 代码+测试 |
| GSM8K | 数学推理 | 高 | 8,500题 | 生成+计算 |

## 使用建议

**全面评估模型能力:**
- 同时使用多个基准,避免单一维度的片面评价
- MMLU + GSM8K + HumanEval 可覆盖知识、推理、代码三大核心能力

**关注模型局限性:**
- TruthfulQA特别重要,揭示模型幻觉问题
- HumanEval和GSM8K测试推理的深度和准确性

**实际应用:**
- 研究论文中通常报告这5个基准
- 用于比较不同模型的相对性能
- 追踪大模型的发展趋势
