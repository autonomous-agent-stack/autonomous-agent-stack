## 核心抽象

### Task (任务)
**定义**: 一次任务实例
**职责**: 派单

**一句话**: task 是派单

### AgentPackage (专业Agent包)
**定义**: 可复制、可分享、可配置的专业 agent 模板
**职责**: 做法

**一句话**: agent package 是做法

### Worker (执行器)
**定义**: 真正干活的执行器
**职责**: 执行腿脚

**一句话**: worker 是执行腿脚

---

## 职责边界

### Task 负责
- 任务输入
- 任务状态
- 任务结果
- 审批状态
- 错误信息

**不负责**:
- ❌ 如何执行 (Worker 负责)
- ❌ 用什么流程 (AgentPackage 负责)
- ❌ 谁来执行 (调度器负责)

### AgentPackage 负责
- 输入/输出 schema
- 能力依赖
- 治理规则(风险/审批/权限)
- 失败处理策略
- 版本和兼容性

**不负责**:
- ❌ 具体执行 (Worker 负责)
- ❌ 任务调度 (调度器负责)
- ❌ 用户交互 (前台入口负责)

### Worker 负责
- 执行任务
- 报告状态
- 返回产物
- 分类错误
- 健康检查

**不负责**:
- ❌ 任务调度 (调度器负责)
- ❌ 审批流程 (control plane 负责)
- ❌ 用户交互 (前台入口负责)

---

## 依赖关系

```
Task
  ├── 依赖: AgentPackage (验证输入/获取执行规则)
  └── 依赖: Worker (执行任务)

AgentPackage
  ├── 依赖: Worker (声明需要什么类型的 worker)
  └── 不依赖: Task (可以被多个任务复用)

Worker
  ├── 依赖: AgentPackage (知道要执行什么)
  └── 不依赖: Task (可以被多个任务复用)
```

---

## 交叉检查

### Q: Task 和 AgentPackage 的区别？
**A**: Task 是"一次执行"，AgentPackage 是"可复用的模板"

类比:
- Task = 一份订单
- AgentPackage = 订单模板

### Q: AgentPackage 和 Worker 的区别？
**A**: AgentPackage 是"做什么"，Worker 是"用什么做"

类比:
- AgentPackage = 菜谱
- Worker = 厨师

### Q: Task 和 Worker 的关系？
**A**: Task 是"派单"，Worker 是"接单干活"

类比:
- Task = 工单
- Worker = 工人

## 禁止事项

### ❌ Task 禁止
- 包含执行逻辑
- 直接调用 Worker
- 存储流程定义

### ❌ AgentPackage 禁止
- 包含状态
- 直接执行任务
- 存储用户数据

### ❌ Worker 禁止
- 做调度决策
- 存储任务定义
- 处理审批

## 生活类比

### 餐厅类比

**Task**: 一张顾客订单
- 内容: 宫保鸡丁 x1, 米饭 x2
- 状态: 待做 / 做中 / 完成
- 结果: 菜端上来了

**AgentPackage**: 菜谱
- 名称: 宫保鸡丁
- 材料: 鸡肉, 花生, 辣椒
- 步骤: 切肉, 炒制, 调味
- 要求: 火候, 时间

**Worker**: 厨师
- 能力: 炒菜, 切菜, 煮饭
- 状态: 空闲 / 忙碌
- 产出: 做好的菜

### 快递类比

**Task**: 一个快递单
- 寄件人: 张三
- 收件人: 李四
- 物品: 书籍
- 状态: 待取 / 运输中 / 已签收

**AgentPackage**: 配送规则
- 类型: 标准快递
- 时效: 3天
- 价格: 10元
- 要求: 包装, 称重

**Worker**: 快递员
- 能力: 骑车, 送货, 签收
- 状态: 空闲 / 配送中
- 区域: 朝阳区
